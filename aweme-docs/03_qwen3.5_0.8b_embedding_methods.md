# 将 Qwen3.5-0.8B 转化为嵌入模型的方法评估

## 1. 问题定义

**目标**：将 Qwen3.5-0.8B（一个混合注意力架构的原生多模态生成模型）转化为高质量的文本嵌入模型。

**挑战**：
- Qwen3.5-0.8B 采用混合注意力架构（75% Gated DeltaNet + 25% Gated Attention），与 Qwen3-Embedding 使用的纯 Transformer 架构不同
- 模型是原生多模态的（`Qwen3_5ForConditionalGeneration`），而非纯文本模型
- 0.8B 规模较小，需要充分利用有限的模型容量

---

## 2. 方法一：EOS Token 池化 + InfoNCE 对比学习（推荐）

### 2.1 方法描述

完全复用 Qwen3-Embedding 的技术路线：
1. 以 Qwen3.5-0.8B 或 Qwen3.5-0.8B-Base 作为基座模型初始化
2. 在输入序列末尾添加 EOS token（`<|endoftext|>`）
3. 从模型最后一层隐藏状态中提取 EOS token 位置的向量作为嵌入
4. 使用 InfoNCE 对比损失进行训练

### 2.2 训练流程

```
阶段 1: 弱监督对比预训练
├── 数据：使用 Qwen3-32B 或其他大模型合成查询-文档对
├── 规模：尽可能大（百万级以上）
├── 损失：InfoNCE（带 batch 内负例）
└── 学习率：较大，如 1e-4

阶段 2: 有监督微调
├── 数据：高质量标注数据（如 MTEB 训练集、STS-B 等）
├── 规模：数十万到数百万对
├── 损失：InfoNCE（带硬负例、QQ/DD 增强、假负例掩码）
└── 学习率：较小，如 1e-5

阶段 3: 模型合并（可选）
└── 使用 SLERP 合并多个微调检查点
```

### 2.3 对混合注意力架构的适配

**EOS Token 池化在 Gated DeltaNet 层中的行为：**

- Gated DeltaNet 层使用固定大小的递归状态 `S_t`，序列信息通过 delta 规则逐步写入状态矩阵
- 当处理到 EOS token 时，状态 `S_t` 已经累积了前面所有 token 的信息
- 因此 EOS token 的输出 `o_t = q̃_t^T · S_t` 确实包含了序列级的语义信息

**结论**：EOS Token 池化与 Gated DeltaNet 机制天然兼容，因为：
- 递归状态机制本身就是信息的逐步累积过程
- EOS 位置的查询向量可以从完整的状态矩阵中提取信息
- 全注意力层（25%）仍然提供全局的 token 间交互

### 2.4 优势与劣势

| 维度 | 评价 |
|------|------|
| **实现复杂度** | 低 - 仅需标准的对比学习训练循环 |
| **与基座模型兼容性** | 高 - 不修改模型架构，保持预训练分布 |
| **与混合注意力兼容性** | 高 - Gated DeltaNet 的递归状态天然支持信息累积 |
| **训练效率** | 高 - 可使用标准的 LoRA/全参数微调 |
| **预期性能** | 中高 - 参考 Qwen3-Embedding-0.6B 的表现 |
| **数据需求** | 中 - 需要大规模对比学习数据 |

## 3. 方法二：Mean Pooling + 对比学习

### 3.1 方法描述

对模型最后一层所有 token 的隐藏状态进行平均池化，得到固定维度的嵌入向量。

```python
# 伪代码
hidden_states = model(input_ids).last_hidden_state  # [batch, seq_len, hidden_dim]
attention_mask = ...  # [batch, seq_len]
embedding = (hidden_states * attention_mask.unsqueeze(-1)).sum(1) / attention_mask.sum(1, keepdim=True)
```

### 3.2 在混合注意力架构中的问题

**因果注意力导致的信息不对称：**

在因果注意力（包括 Gated DeltaNet 和 Gated Attention）中：
- 第 1 个 token 只能看到自己
- 第 2 个 token 能看到第 1、2 个 token
- 第 n 个 token 能看到第 1 到第 n 个 token

因此，前面 token 的表示缺乏后续 token 的信息，平均池化会稀释序列末尾 token 携带的完整信息。

**Gated DeltaNet 层的特殊情况：**

在 Gated DeltaNet 层中，由于门控衰减机制，较早的 token 信息可能已经被后续 token 部分覆盖。Mean Pooling 可能会引入这些已被"遗忘"的早期 token 的不完整表示。

### 3.3 优势与劣势

| 维度 | 评价 |
|------|------|
| **实现复杂度** | 低 |
| **与基座模型兼容性** | 中 - 因果注意力下效果可能受限 |
| **与混合注意力兼容性** | 低 - Gated DeltaNet 的衰减机制使早期 token 表示不完整 |
| **训练效率** | 高 |
| **预期性能** | 中低 - 可能不如 EOS 池化 |
| **数据需求** | 中 |

## 4. 方法三：双向注意力改造 + Mean/CLS 池化

### 4.1 方法描述

修改模型的注意力掩码，将因果注意力（下三角掩码）替换为双向注意力（全连接掩码），使每个 token 都能关注到序列中的所有其他 token。

```python
# 修改注意力掩码
# 原始：causal_mask (下三角矩阵)
# 修改后：full_mask (全 1 矩阵，仅 padding 位置为 0)
```

### 4.2 在混合注意力架构中的巨大挑战

**全注意力层（25%）**：理论上可以直接移除因果掩码，变为双向注意力。但这会偏离预训练时的注意力模式。

**Gated DeltaNet 层（75%）**：**这是最大的问题**。Gated DeltaNet 本质上是一个递归模型：

```
S_t = g_t · S_{t-1} + k̃_t ⊗ [β_t · (v_t − (g_t · S_{t-1})^T k̃_t)]
```

状态 `S_t` 是按时间顺序逐步计算的，天然是因果的。要实现"双向" Gated DeltaNet，有两种可能：

1. **双向扫描**：分别进行前向和后向扫描，然后合并结果（类似 BiMamba）
   - 需要大幅修改模型架构
   - 计算量翻倍
   - 预训练权重无法直接复用于后向部分

2. **放弃线性注意力的因果性**：改为使用分块并行的全连接形式
   - 破坏了线性注意力的效率优势
   - 等价于退化为标准注意力

### 4.3 优势与劣势

| 维度 | 评价 |
|------|------|
| **实现复杂度** | 极高 - 需要修改 75% 的层的核心计算逻辑 |
| **与基座模型兼容性** | 低 - 大幅偏离预训练分布 |
| **与混合注意力兼容性** | 极低 - Gated DeltaNet 天然因果，无法简单改为双向 |
| **训练效率** | 低 - 需要大量额外训练来弥补分布偏移 |
| **预期性能** | 不确定 - 可能需要大量数据才能收敛 |
| **数据需求** | 高 - 需要大规模训练来适应新的注意力模式 |

**结论：不推荐此方法用于 Qwen3.5。** Gated DeltaNet 的递归本质使得双向注意力改造几乎不可行。

## 5. 方法四：多阶段训练流水线（工程化方案）

### 5.1 方法描述

结合方法一的 EOS 池化策略，设计完整的工程化训练流水线：

```
阶段 0: 继续预训练（可选）
├── 在目标领域的大规模无标注文本上继续训练
├── 使用标准的因果语言模型损失
└── 目的：适配目标领域的分布

阶段 1: 无监督对比学习
├── 使用无标注文本对构建正例对
│   ├── 相邻句子对
│   ├── 同一文档的标题-段落对
│   └── 翻译对（多语言场景）
├── 使用 SimCSE 风格的 dropout 增强
└── InfoNCE 损失

阶段 2: 有监督对比学习
├── 使用高质量标注数据
├── 支持硬负例
├── InfoNCE + QQ/DD 增强
└── 可选：知识蒸馏（从更大的嵌入模型蒸馏）

阶段 3: 任务特化微调（可选）
├── 针对特定下游任务微调
├── 使用更少但更精确的数据
└── 较小的学习率

阶段 4: 模型合并
└── SLERP 合并多个检查点
```

### 5.2 优势与劣势

| 维度 | 评价 |
|------|------|
| **实现复杂度** | 中 - 多阶段但每个阶段标准 |
| **与基座模型兼容性** | 高 - 不修改架构 |
| **与混合注意力兼容性** | 高 - 与方法一相同 |
| **训练效率** | 中 - 多阶段需要更多时间 |
| **预期性能** | 高 - 多阶段训练通常效果最好 |
| **数据需求** | 高 - 每个阶段都需要数据 |

## 6. 方法五：加权层池化（Weighted Layer Pooling）

### 6.1 方法描述

不仅使用最后一层的隐藏状态，而是对模型多层的隐藏状态进行加权组合：

```python
# 伪代码
all_hidden_states = model(input_ids, output_hidden_states=True).hidden_states
# all_hidden_states: tuple of (num_layers + 1) tensors, each [batch, seq_len, hidden_dim]

# 可学习的层权重
layer_weights = nn.Parameter(torch.ones(num_layers + 1))
weights = F.softmax(layer_weights, dim=0)

# 加权组合
weighted_output = sum(w * h for w, h in zip(weights, all_hidden_states))

# 然后对加权输出做 EOS 池化或 Mean 池化
embedding = weighted_output[:, -1, :]  # EOS position
```

### 6.2 在混合注意力架构中的考虑

- 不同层的类型不同（Gated DeltaNet vs Gated Attention），其隐藏状态的分布可能差异较大
- 可学习的层权重可以自适应地选择最有用的层信息
- 可能发现全注意力层的隐藏状态更适合做嵌入

### 6.3 优势与劣势

| 维度 | 评价 |
|------|------|
| **实现复杂度** | 低 - 仅增加少量可学习参数 |
| **与基座模型兼容性** | 高 - 不修改核心架构 |
| **与混合注意力兼容性** | 中 - 需要观察不同层类型的贡献 |
| **训练效率** | 中 - 需要输出所有层的隐藏状态，内存开销增大 |
| **预期性能** | 中 - 可能小幅提升 |
| **数据需求** | 中 |

## 8. 推荐方案

### 8.1 首选方案：方法一 + 方法四的组合

**推荐采用 EOS Token 池化 + 多阶段训练流水线**，具体如下：

```
1. 基座模型初始化：Qwen3.5-0.8B-Base（或 Qwen3.5-0.8B）

2. 嵌入提取：EOS Token 池化
   - 输入格式: "{Instruction} {Query}<|endoftext|>"
   - 嵌入向量 = last_hidden_state[EOS_position]

3. 训练流程：
   阶段 1: 大规模弱监督对比学习
   ├── 数据: 使用 LLM 合成的查询-文档对（百万级）
   ├── 损失: InfoNCE (τ=0.1)
   ├── 训练方式: 全参数微调或 LoRA
   └── Epoch: 1-3

   阶段 2: 高质量有监督微调
   ├── 数据: 标注数据 + 筛选合成数据
   ├── 损失: InfoNCE + 硬负例 + QQ/DD 增强
   ├── 训练方式: 全参数微调
   └── Epoch: 1-2

   阶段 3: SLERP 模型合并

4. 训练超参考值：
   ├── 学习率: 阶段1 1e-4, 阶段2 1e-5
   ├── Batch Size: 尽可能大（提供更多 in-batch 负例）
   ├── 温度: 0.1
   ├── 最大长度: 512-1024（嵌入任务通常不需要太长）
   └── DeepSpeed: ZeRO-2
```

### 8.2 理由

1. **最小化架构修改风险**：不修改模型架构，完全保持预训练分布
2. **与 Gated DeltaNet 天然兼容**：递归状态累积机制支持 EOS 位置的信息汇聚
3. **经过验证的方法论**：Qwen3-Embedding 论文已在更大模型上验证了这一方法的有效性
4. **ms-swift 框架已有部分支持**：框架已实现 InfoNCE 损失、EmbeddingTrainer 等关键组件

### 8.3 Qwen3.5 混合架构的特殊优势

相比纯 Transformer 模型，Qwen3.5-0.8B 作为嵌入模型可能有以下优势：

1. **更高效的长文本嵌入**：Gated DeltaNet 层的 O(n) 复杂度使得处理长文本更加高效
2. **固定大小的状态表示**：线性注意力层的递归状态天然提供了一种序列压缩表示
3. **多粒度的信息捕获**：全注意力层捕获全局信息，线性注意力层捕获局部演化信息

## 9. 需要注意的风险

1. **模态冲突**：Qwen3.5-0.8B 是多模态模型（包含视觉编码器），如果仅用于文本嵌入，视觉部分的参数可能造成浪费或干扰。可能需要：
   - 冻结视觉编码器
   - 或者只使用文本部分的参数

2. **小模型容量限制**：0.8B 的参数量较小，可能需要更精心的数据策略和训练超参数调优

3. **框架支持**：当前 ms-swift 的 qwen3point5 分支尚未为 Qwen3.5 注册专门的嵌入模型类型和模板（详见文档四的分析）

4. **Gated DeltaNet 的嵌入质量**：虽然理论上 EOS 池化与递归状态兼容，但 Gated DeltaNet 的门控衰减可能导致早期信息被遗忘。实际效果需要通过实验验证
