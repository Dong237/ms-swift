# 多Teacher GKD 升级报告

> 本文档记录 `multi-teacher-upgrade` 分支对多Teacher GKD 实现的系统性升级，包括方法论、逐批次改动内容以及完整的 commit 追踪。

---

## 1. 升级背景

原始多Teacher GKD 实现（`multi-teacher` 分支，3个commit）完成了核心功能：基于 `channel` 字段的按样本路由、Teacher去重加载、DeepSpeed/FSDP支持。但经全面审查后发现 **16个问题**，涵盖静默正确性Bug、缺失的运行时防护、测试覆盖不足、UX不友好以及功能缺失。

本次升级采用 **分批次递进** 策略，按优先级从高到低修复，每批次独立可回退。

---

## 2. 升级方法论

### 2.1 问题识别

对 `gkd_trainer.py`、`rlhf_args.py`、`rlhf.py` 进行逐行审查，识别出16个问题并分为5类：

| 类别 | 问题数量 | 严重程度 |
|------|---------|---------|
| 静默正确性Bug | 3 | 高 — 产生错误梯度但无报错 |
| 缺失运行时防护 | 3 | 中高 — 特定配置下导致死锁或加载错误 |
| 测试覆盖不足 | 4 | 中 — 关键路径无测试验证 |
| UX改进 | 3 | 低 — 不影响正确性但影响可用性 |
| 功能缺失 | 2 | 低 — 增强功能但非核心 |

### 2.2 批次化策略

按 **依赖关系 + 严重程度** 排列，确保：
- 每个批次独立可测试
- 高优先级Bug先修
- 后续批次可依赖前序批次的修复

### Batch 5: 按Teacher独立超参（per-teacher beta / temperature）

**Commit:** `31f1cbcd` — `feat: per-teacher beta and temperature for multi-teacher GKD`

**功能描述：** 允许为每个channel指定独立的 JSD loss 超参数（beta 和 temperature），不同领域的Teacher可以使用不同的蒸馏策略。

**新增CLI参数：**

| 参数 | 类型 | 描述 |
|------|------|------|
| `--teacher_beta_map` | JSON字符串 | 每个channel的beta值映射，如 `'{"math": 0.8, "code": 0.3}'` |
| `--teacher_temperature_map` | JSON字符串 | 每个channel的temperature值映射，如 `'{"math": 0.7, "code": 1.5}'` |

**实现细节：**
- 参数解析与验证：`_check_gkd()` 中解析JSON、校验key子集关系、beta范围[0,1]、temperature>0
- 新增 `_compute_grouped_jsd_loss()` 方法：
  1. 通过mask的每个样本token数，将flat token index映射回样本的channel
  2. 按 `(beta, temperature)` 唯一对分组
  3. 对每组独立计算 `generalized_jsd_loss`
  4. 按token数加权平均
- `compute_loss()` 在检测到per-teacher超参时路由到grouped loss，否则走原有路径

**新增测试：** 7个参数解析测试 + 5个grouped loss单元测试 + 2个集成测试

### Batch 5 补丁: Liger兼容性修复

**Commit:** `dc2dd0c1` — `fix: skip Liger path when per-teacher beta/temperature is active`

**问题：** Liger fused JSD loss 在初始化时固定了全局 beta/temperature，无法感知per-teacher超参。当Liger启用且批次内只有单一teacher时，Liger路径会短路绕过per-teacher超参检查。

**修复：** 在Liger路径入口添加 `not has_per_teacher_params` 条件，确保per-teacher超参生效时回退到标准路径。

## 5. 已知遗留问题

以下问题在审查中识别但未在本次升级中修复（属于预存问题或低优先级）：

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| 标准路径未传递 `--temperature` 给 `generalized_jsd_loss` | 中 | 预存Bug，非Liger路径的temperature始终为1.0。Liger路径正常使用temperature |
| `channel` 字段泄漏到 `model.generate()` | 低 | 预存问题，HF模型通过 `**kwargs` 静默忽略，但不够干净 |
| Megatron后端不支持多Teacher | 低 | 已添加 `NotImplementedError` 明确拒绝，非本次升级范围 |
