# 多Teacher GKD 使用指南

> 本文档是多Teacher GKD 的完整使用手册，涵盖原理介绍、数据准备、参数配置、训练脚本示例以及常见问题排查。

---

## 1. 什么是多Teacher GKD

### 1.1 GKD 简介

GKD（Generalized Knowledge Distillation）通过最小化学生模型和教师模型输出分布之间的 **广义JSD散度** 进行知识蒸馏：

```
D_JSD(β)(P_teacher, P_student) = β·KL(P_teacher || M) + (1-β)·KL(P_student || M)
```

其中 `M = β·P_teacher + (1-β)·P_student` 是混合分布。

### 1.2 多Teacher扩展

标准GKD使用单个Teacher。多Teacher GKD 允许不同领域的样本路由到不同的专家Teacher模型：

```
                    ┌─────────────────────┐
                    │     训练数据          │
                    │  (含 channel 字段)    │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │     Channel 路由     │
                    │  math → teacher_0   │
                    │  code → teacher_1   │
                    │  general → teacher_2│
                    └──┬──────┬──────┬────┘
                       │      │      │
              ┌────────▼──┐ ┌─▼────┐ ┌▼─────────┐
              │ 数学Teacher│ │代码  │ │ 通用     │
              │ (frozen)  │ │Teacher│ │ Teacher  │
              └────────┬──┘ └─┬────┘ └┬─────────┘
                       │      │       │
                  soft labels ×3 (per-sample)
                       │      │       │
                    ┌──▼──────▼───────▼──┐
                    │  JSD Loss (per-sample)│
                    │  支持per-teacher超参   │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   Student模型 (可训练) │
                    └─────────────────────┘
```

### 1.3 核心特性

- **按样本路由**：批次内每个样本独立路由到其指定Teacher
- **Teacher去重**：多个channel指向同一模型路径时只加载一次
- **Per-teacher超参**：每个channel可独立设置 beta 和 temperature
- **三种训练模式**：Off-policy（数据集响应）、On-policy（学生生成）、Seq-KD（Teacher生成）
- **完全向后兼容**：单Teacher GKD 行为不变
- **vLLM支持**：学生的 on-policy 生成可使用vLLM加速

## 2. 数据准备

### 2.1 数据格式

每条训练数据需包含 `channel` 字段，对应 `--teacher_domain_map` 中的某个key：

```jsonl
{"messages":[{"role":"user","content":"求解 2x + 3 = 7"},{"role":"assistant","content":"x = 2"}],"channel":"math"}
{"messages":[{"role":"user","content":"写一个Python排序函数"},{"role":"assistant","content":"def sort(arr): return sorted(arr)"}],"channel":"code"}
{"messages":[{"role":"user","content":"介绍一下气候变化"},{"role":"assistant","content":"气候变化是指..."}],"channel":"general"}
```

### 2.2 注意事项

- `channel` 字段在 ms-swift 数据管线中是一等字段，自动流经预处理器、模板编码器和 data_collator
- **没有 `channel` 字段的样本默认路由到 teacher[0]**（domain map 中第一个唯一Teacher）
- 可以在同一数据集中混合有/无 channel 的样本
- `channel` 字段只影响Teacher路由，不影响训练消息内容

## 3. CLI参数

### 3.1 核心参数

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `--teacher_domain_map` | JSON字符串 | None | channel名到Teacher模型路径的映射 |
| `--teacher_model` | 字符串 | None | 单Teacher路径（与 `teacher_domain_map` 互斥） |
| `--teacher_type_map` | JSON字符串 | None | channel名到模型类型的映射（用于微调模型的类型识别） |

### 3.2 Per-teacher 超参数

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `--teacher_beta_map` | JSON字符串 | None | 每个channel的beta值。beta控制JSD中Teacher权重：越大越偏向Teacher分布 |
| `--teacher_temperature_map` | JSON字符串 | None | 每个channel的temperature值。越大分布越平滑 |
| `--beta` | float | 0.5 | 全局beta（未在map中指定的channel使用此值） |
| `--temperature` | float | 0.9 | 全局temperature（未在map中指定的channel使用此值） |

**Per-teacher超参规则：**
- `teacher_beta_map` 的 key 必须是 `teacher_domain_map` 的子集
- beta值范围：[0, 1]（0=forward KL，0.5=对称JSD，1=reverse KL）
- temperature值范围：> 0
- 未在map中指定的channel自动使用全局 `--beta` / `--temperature`

### 3.3 训练模式参数

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `--seq_kd` | bool | False | Teacher生成响应替代数据集响应 |
| `--lmbda` | float | 0.5 | On-policy概率：以此概率使用学生生成的响应 |
| `--sft_alpha` | float | 0 | 辅助SFT loss权重（仅对数据集/Teacher数据生效） |

### 3.4 内存优化参数

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `--offload_teacher_model` | bool | False | Teacher在forward pass之间卸载到CPU |
| `--teacher_deepspeed` | 字符串 | None | Teacher专用DeepSpeed配置（如 `zero2`） |
| `--use_liger_kernel` | bool | False | 使用Liger fused JSD loss（单Teacher批次时生效） |

## 4. 训练脚本示例

### 4.1 基础多Teacher训练（单节点8卡）

最简配置，off-policy 训练（使用数据集中的响应）：

```bash
NPROC_PER_NODE=8 \
swift rlhf \
    --rlhf_type gkd \
    --model Qwen/Qwen2.5-7B \
    --teacher_domain_map '{"math":"/data/models/math-teacher","code":"/data/models/code-teacher"}' \
    --dataset /data/train.jsonl \
    --deepspeed zero2 \
    --num_train_epochs 3 \
    --per_device_train_batch_size 4 \
    --gradient_accumulation_steps 4 \
    --learning_rate 5e-6 \
    --lr_scheduler_type cosine \
    --warmup_ratio 0.05 \
    --beta 0.5 \
    --temperature 0.9 \
    --max_length 4096 \
    --logging_steps 10 \
    --save_strategy epoch \
    --output_dir /data/output/multi_teacher_gkd
```

### 4.2 Per-teacher超参 + Teacher offload

不同领域使用不同蒸馏强度（数学用高beta偏向Teacher，代码用低beta偏向学生）：

```bash
NPROC_PER_NODE=8 \
swift rlhf \
    --rlhf_type gkd \
    --model Qwen/Qwen2.5-7B \
    --teacher_domain_map '{"math":"/data/models/math-72B","code":"/data/models/code-72B"}' \
    --teacher_beta_map '{"math": 0.9, "code": 0.3}' \
    --teacher_temperature_map '{"math": 0.7, "code": 1.2}' \
    --dataset /data/train.jsonl \
    --offload_teacher_model true \
    --deepspeed zero2 \
    --num_train_epochs 3 \
    --per_device_train_batch_size 2 \
    --gradient_accumulation_steps 8 \
    --learning_rate 5e-6 \
    --max_length 4096 \
    --output_dir /data/output/per_teacher_beta
```

### 4.3 指定Teacher模型类型（微调模型）

当Teacher是微调检查点无法自动识别类型时：

```bash
swift rlhf \
    --rlhf_type gkd \
    --model Qwen/Qwen2.5-7B \
    --teacher_domain_map '{"math":"/data/ft-models/math-qwen3","code":"/data/ft-models/code-qwen3"}' \
    --teacher_type_map '{"math":"qwen3","code":"qwen3"}' \
    --dataset /data/train.jsonl \
    --deepspeed zero2 \
    --num_train_epochs 3 \
    --per_device_train_batch_size 4 \
    --learning_rate 5e-6 \
    --output_dir /data/output/typed_teachers
```

部分指定也可以——省略的domain自动检测类型：

```bash
--teacher_domain_map '{"math":"/data/ft-models/math-qwen3","code":"deepseek-ai/DeepSeek-V3"}'
--teacher_type_map '{"math":"qwen3"}'
# code域使用Hub模型，自动检测为 deepseek_v3
```

### 4.4 Seq-KD模式（Teacher生成响应）

```bash
NPROC_PER_NODE=8 \
swift rlhf \
    --rlhf_type gkd \
    --model Qwen/Qwen2.5-7B \
    --teacher_domain_map '{"math":"/data/models/math-teacher","code":"/data/models/code-teacher"}' \
    --dataset /data/train.jsonl \
    --seq_kd true \
    --max_completion_length 512 \
    --deepspeed zero2 \
    --num_train_epochs 3 \
    --per_device_train_batch_size 4 \
    --learning_rate 5e-6 \
    --output_dir /data/output/seq_kd
```

多Teacher seq_kd 使用 **per-teacher生成**：每个样本由其指定的Teacher生成响应（而非多数Teacher），保证响应和soft label语义一致。

### 4.5 On-policy + vLLM（colocate模式）

```bash
NPROC_PER_NODE=4 \
swift rlhf \
    --rlhf_type gkd \
    --model Qwen/Qwen2.5-7B \
    --teacher_domain_map '{"math":"/data/models/math-teacher","code":"/data/models/code-teacher"}' \
    --dataset /data/train.jsonl \
    --use_vllm true \
    --vllm_mode colocate \
    --lmbda 0.5 \
    --max_completion_length 512 \
    --deepspeed zero2 \
    --num_train_epochs 3 \
    --per_device_train_batch_size 4 \
    --learning_rate 5e-6 \
    --output_dir /data/output/on_policy_vllm
```

vLLM仅负责学生的on-policy生成，与Teacher推理完全无关。

### 4.6 On-policy + vLLM（server模式，单节点）

将8张GPU分为6张训练 + 2张vLLM server，释放训练GPU内存：

```bash

    --model /data/models/Qwen2.5-7B \
    --vllm_max_model_len 4096 \
    --vllm_tensor_parallel_size 2 \
```

## 5. 路由机制详解

### 5.1 路由流程

1. **数据加载**：每个样本的 `channel` 字段通过数据管线保留
2. **批次组装**：标准batching，不需要按channel排序（但排序可提升Liger效率）
3. **`compute_loss()` 内部**：
   - `channel` 字段从inputs中pop出，防止泄漏到模型forward
   - `_get_teacher_indices()` 将channel映射到teacher索引
   - 若批次内所有样本用同一teacher → 单teacher forward（快速路径）
   - 若混合teacher → `_multi_teacher_forward()` 分组forward后scatter回
4. **Loss计算**：
   - 无per-teacher超参 → 统一调用 `generalized_jsd_loss`
   - 有per-teacher超参 → `_compute_grouped_jsd_loss` 按 `(beta, temp)` 分组计算
5. **词表对齐**：不同词表大小的Teacher自动对齐，填充位使用 `-1e9`（softmax后概率≈0）

### 5.2 Liger Kernel 行为

Liger fused JSD loss（`--use_liger_kernel true`）仅在以下条件全部满足时使用：
- 批次内只有单一teacher
- 未设置per-teacher超参（`teacher_beta_map` / `teacher_temperature_map`）

其他情况自动回退到标准路径，并输出一次性警告。

## 6. 常见问题

### `teacher_domain_map and teacher_model are mutually exclusive`
`--teacher_model` 和 `--teacher_domain_map` 不能同时设置。多Teacher用 `teacher_domain_map`，单Teacher用 `teacher_model`。

### `GKD requires either --teacher_model or --teacher_domain_map`
GKD训练必须指定至少一个Teacher来源。

### `teacher_domain_map must be a non-empty JSON dict`
JSON格式错误或为空。确保是有效的JSON字符串且至少有一个条目。

### `ZeRO-3 for teacher models is incompatible with multi-teacher per-sample routing`
Teacher的DeepSpeed配置不能用ZeRO-3（stage 3），因为不同rank可能调用不同teacher导致AllGather死锁。解决方案：使用 `--teacher_deepspeed zero2`。注意：student的DeepSpeed可以用ZeRO-3，仅teacher不行。

### `teacher_beta_map key "X" not found in teacher_domain_map`
`teacher_beta_map` 或 `teacher_temperature_map` 中的key必须存在于 `teacher_domain_map`。检查是否有拼写错误。

### `teacher_beta_map requires --teacher_domain_map`
Per-teacher超参只能与多Teacher模式（`teacher_domain_map`）一起使用。

### 样本没有 `channel` 字段
无 `channel` 字段的样本自动路由到 teacher[0]（domain map中第一个唯一Teacher）。可以混合有/无channel的数据。

### 不同Teacher词表大小
自动处理。Teacher logits被填充到最大词表大小对齐，填充位使用 `-1e9` 保证softmax后概率接近0。

### Teacher模型类型自动检测失败
微调检查点可能缺少可识别的 `architectures` 字段。使用 `--teacher_type_map` 显式指定。常见值：`qwen3`、`qwen2`、`llama`、`deepseek_v3`、`glm4`、`mistral`。

## 7. 局限性

| 限制 | 说明 |
|------|------|
| Megatron后端不支持 | 多Teacher仅支持 HuggingFace/DeepSpeed/FSDP 训练路径。Megatron会抛出 `NotImplementedError` |
| Padding-free / Packing 不兼容 | channel元数据是per-sample的，packing时无法对应。多Teacher训练时请禁用 `--padding_free` 和 `--packing` |
| Teacher的DeepSpeed不能用ZeRO-3 | 多Teacher按样本路由与ZeRO-3的AllGather同步冲突。student可以用ZeRO-3 |
| Liger kernel + per-teacher超参不兼容 | Liger loss在初始化时固定全局beta/temperature，有per-teacher超参时自动回退到标准路径 |
| Teacher不支持vLLM推理 | GKD需要完整logits分布，vLLM API不返回full logits。Teacher始终使用PyTorch forward |

## 8. 单Teacher vs 多Teacher 对比

| 方面 | 单Teacher | 多Teacher |
|------|----------|----------|
| CLI | `--teacher_model path` | `--teacher_domain_map '{"ch":"path",...}'` |
| 模型类型 | `--teacher_model_type type` | `--teacher_type_map '{"ch":"type",...}'` |
| 数据 | 标准messages | messages + `channel` 字段 |
| 超参 | 全局 `--beta` / `--temperature` | 可选 `--teacher_beta_map` / `--teacher_temperature_map` |
| Teacher加载 | 1个模型 | N个唯一模型（自动去重） |
| Loss | JSD（所有样本 vs 1个Teacher） | JSD（每个样本 vs 其路由的Teacher，支持per-teacher超参） |
| seq_kd | 1个Teacher生成 | 每个Teacher为其样本独立生成 |
| vLLM | 学生生成 | 学生生成（不变） |
| Offloading | 1个Teacher来回CPU | 每个Teacher独立来回CPU |
